2) One-page print every time (isolated window CSS)

You’re already using the isolated-print helper in Laboratory.tsx — great. Tweak its inline CSS so the content fits inside A4 with internal padding (no overflow → no page 2):

Replace the CSS string inside printIsolated() with this:

const css = `
  <style>
    @page { size: A4; margin: 0; }              /* browser adds no extra margins */
    html, body { margin: 0; padding: 0; }
    .rx-print {
      width: 210mm;
      height: 297mm;                            /* exact page box */
      padding: 12mm;                            /* inner margins */
      box-sizing: border-box;
      display: flex;
      flex-direction: column;
    }
    .mt-auto { margin-top: auto !important; }   /* pin footer to bottom */
    /* Safety: if content is just a tad tall, gently scale to fit one page */
    .fit { transform: scale(0.985); transform-origin: top left; }
  </style>
`;


This prevents the tiny height spill you were seeing (previous CSS used min-height and page margins that could add up to >297 mm). Your current Lab code already uses this isolated print approach; we’re just tightening it. 

Tip: You don’t need any global @media print to print Lab now. The isolated window brings its own CSS and won’t be affected by the app’s layout. Your PDFs confirm the isolated print flow is working; we just ensure perfect sizing.
